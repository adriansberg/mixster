## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-background` (dark: oklch 25.45%) | Page background — always dark mode for party context |
| Secondary (30%) | `--color-card` / `bg-card/50` | Playlist cards, section containers, device selector panel |
| Accent (10%) | Gradient: purple-600 → pink-500 → orange-400 | Enabled/selected state only (see reserved list below) |
| Destructive | `--color-destructive` (oklch 46.04% dark) | "Bekreft?" confirm state on TØM HISTORIKK only |
| Muted | `--color-muted-foreground` | Track counts, helper text, empty-state copy |
| Border | `--color-border` | Card borders, dividers |

Accent gradient (`bg-linear-to-r from-purple-600 via-pink-500 to-orange-400`) reserved for:
1. **Enabled/selected default playlist toggle cards** — active state background
2. **Enabled custom playlist toggle cards** — active state border highlight + background (D-07)
3. **START SPILL button** — primary CTA, always gradient
4. **NESTE SANG / VIS SANG buttons** — game action CTAs
5. **Year badge pill** on flip card front
6. **Card back** of the flip card (hidden song state)

Inactive/disabled state: `bg-card hover:bg-accent` (secondary surface, no gradient)

## Component States

### Playlist Toggle Card (default + custom)

| State | Classes |
|-------|---------|
| Enabled/selected | `bg-linear-to-br from-purple-600 to-pink-500 text-white border-purple-400 shadow-lg` |
| Disabled/unselected | `bg-card hover:bg-accent border` |
| Touch feedback | `hover:scale-105 active:scale-95 transition-all` |

Source: CONTEXT.md D-07 + existing setup page toggle classes

### TØM HISTORIKK Button — Double-Press Pattern

| State | Text | Styling |
|-------|------|---------|
| Idle | "TØM HISTORIKK" | `variant="outline"` (default) |
| Pending confirmation | "Bekreft?" | `variant="destructive"` — red background |
| Post-clear (2s) | "Slettet!" | `variant="outline"` + muted — then reverts |

Interaction contract (CONTEXT.md D-12, D-13):
- First click: sets `clearPending = true`, button shows "Bekreft?" in destructive red
- Second click (while pending): executes clear, shows "Slettet!" for 2s, resets
- Click anywhere else: resets `clearPending` to false (use `onclick:outside` or `onblur`)
- No `window.confirm()` anywhere

### Songs-Played Counter (CONTEXT.md D-10, D-11)

- Location: play page header bar, adjacent to "shitster" logo
- Format: `"X sanger spilt"` where X is the count
- Size: `text-sm` (14px), `text-muted-foreground`, weight 400
- Visible at all times during play — TV-readable distance requires adequate contrast

### Custom Playlist "Fjern" Button (CONTEXT.md D-08)

- Always visible — never hidden on hover
- `variant="ghost"` `size="sm"` — small, non-destructive appearance
- No confirmation required (non-destructive relative to game session)

## Interaction Contracts

### localStorage Schema Migration (CONTEXT.md D-01–D-04)

No visible UI change. On load, silently migrate old keys or reset to defaults. User never sees a migration prompt.

### Playlist Enable/Disable Toggle Persistence (CONTEXT.md D-05–D-06)

Write to `shitster_playlists` on every toggle — no deferred save. No loading state needed (localStorage write is synchronous).

### START SPILL Disabled Guard (CONTEXT.md D-09)

Button is disabled when `(enabledDefaults.length + enabledCustoms.length) === 0`. Disabled styling via `disabled` prop (bits-ui Button handles opacity/cursor). No error message required — the visual state (no active cards) communicates the problem.

### History Clear Location (CONTEXT.md D-14–D-15)

TØM HISTORIKK appears on both play page (existing header + mobile bar) and setup page (new — add below custom playlists section or in a session management section). Identical double-press UX on both.
